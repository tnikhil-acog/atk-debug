## Core Philosophy

`atk-debug` is a **drop-in replacement** for the `debug` package. It preserves the familiar namespace-based logging pattern while adding features that solve real-world debugging challenges, such as trace visibility and production-only safeguards.

## Automatic Caller Metadata

By default, every log generated by `atk-debug` includes the file name and line number where the log was called.

```ts
const log = atkDebug('app:module');
log('Initializing...'); 
// Output: app:module [main.ts:5] Initializing...
```

### Why is this useful?
- No more searching for text strings in your codebase.
- Instant context for logs coming from shared utility functions.
- works across callsites and even within wrapped functions.

### Performance Tuning
If you are logging in a tight loop and performance is a priority, you can disable metadata capture:

```ts
atkDebug.setCallerMetaEnabled(false);
// ... performance critical path ...
atkDebug.setCallerMetaEnabled(true);
```

## Namespaces and Extension

Namespaces help you filter logs. You can create sub-namespaces easily:

```ts
const api = atkDebug('app:api');
const users = api.extend('users'); // Becomes 'app:api:users'

users('Creating user...');
```

Enable them via the environment variable:
```bash
DEBUG=app:api:* bun run dev
```

## Runtime Configuration

`atk-debug` provides several methods to control logging behavior programmatically.

### `setDevOnly(boolean)`
When `true` (default), logs will only appear if `NODE_ENV` is not `production`. This prevents accidental log leakage in production environments.

```ts
atkDebug.setDevOnly(true); // Logs hidden in production node environments
```

### `enable(namespaces)` / `disable()`
Just like the core `debug` package, you can toggle namespaces at runtime.

```ts
atkDebug.enable('app:*, -app:secret'); 
```

## Advanced Usage

### Custom Formatters
Add custom formatters to handle specific object types:

```ts
atkDebug.formatters.h = (v) => {
  return v.toString('hex');
};

log('Buffer contents: %h', Buffer.from('hello'));
```

### Redirecting Output
You can override where logs are written (e.g., to a file or a different stream):

```ts
import fs from 'fs';
const stream = fs.createWriteStream('./app.log');

atkDebug.log = (...args) => {
  stream.write(args.join(' ') + '\n');
};
```

## Best Practices

1. **Hierarchy Matters**: Use colon-separated namespaces (`service:auth:db`) to allow granular filtering.
2. **Objects vs. Strings**: Pass objects as separate arguments to leverage the built-in inspector (`log('data', myObj)` instead of `log('data ' + myObj)`).
3. **Environment First**: Prefer using the `DEBUG` environment variable over programmatic `enable()` for flexibility.
4. **Metadata usage**: Keep caller metadata enabled during development, but consider disabling it if you notice performance bottlenecks in extremely high-frequency logging paths.
